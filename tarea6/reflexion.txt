El ejercicio permitió comprender de manera práctica cómo se asegura la comunicación entre microservicios mediante JWT y cómo Redis se utiliza como capa de control para validar y revocar tokens en tiempo real. En lugar de depender de una base de datos persistente para las sesiones, Redis ofrece una respuesta inmediata para la validación de cada petición, reforzando la escalabilidad y el rendimiento del sistema.
También se evidenció la importancia de CORS para habilitar la interacción entre servicios y el cliente web, especialmente en entornos distribuidos o en nubes híbridas.
Durante la implementación, se destacó el uso de logs para rastrear solicitudes, detectar errores y analizar la duración de cada operación. Esta trazabilidad es esencial para el mantenimiento y la observabilidad de sistemas modernos.
El cliente web desarrollado permitió visualizar de manera tangible el ciclo de vida de los tokens —emisión, expiración, refresco y revocación— y contrastar la información local con el estado real almacenado en Redis.
Finalmente, este proyecto consolidó la comprensión de los principios de seguridad, aislamiento y escalabilidad en una arquitectura basada en microservicios, demostrando cómo herramientas ligeras pueden emular comportamientos de sistemas distribuidos reales.
